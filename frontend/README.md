# Frontend

This directory contains the React + TypeScript + Vite demo app for the TicketNFTs project. It is a local-facing interface for exercising the smart contract during development and demos.

## What Exists Today

- `src/App.tsx` renders a single-page "control room" for the local demo flow.
- `src/contract.ts` connects to MetaMask with `ethers`, loads the deployed contract address from `frontend/.env.local`, and exposes wallet/contract helpers.
- The UI currently supports:
  - connecting a wallet
  - owner-only minting
  - fetching ticket info and current owner
  - redeeming a ticket once
  - transferring a ticket before redemption
- The deployed contract address and expected chain ID are read from:
  - `VITE_TICKET_NFT_ADDRESS`
  - `VITE_TICKET_NFT_CHAIN_ID`

## What This Frontend Assumes

- MetaMask (or another injected EIP-1193 wallet) is available in the browser.
- A `TicketNFT` contract has already been deployed.
- `frontend/.env.local` has been generated by the root deployment script.

## Local Run Instructions

From the repo root:

```bash
npm install
cd frontend
npm install
```

Start the local chain in one terminal from the repo root:

```bash
npx hardhat node
```

Deploy the contract in a second terminal from the repo root:

```bash
npm run deploy:local
```

That deploy step writes `frontend/.env.local` automatically. Then start the frontend:

```bash
cd frontend
npm run dev
```

## Demo Flow

1. Connect MetaMask to the local Hardhat network and import a funded local account if needed.
2. Connect the deployer wallet to mint a ticket.
3. Use the token ID to inspect the stored ticket data.
4. Transfer the ticket to another account before redemption if you want to test ownership changes.
5. Redeem the ticket as the current owner.
6. Verify that transferring the redeemed ticket now fails.
